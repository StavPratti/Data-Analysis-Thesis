Airbnb Data Analysis

My thesis focuses on the analysis and visualization of data extracted from the Airbnb platform, specifically related to properties across four major regions of Greece: Athens, Thessaloniki, Crete, and the islands of the Southern Aegean. The result of this work is the design of two distinct dashboards: one strategic dashboard for the Ministry of Finance and one operational dashboard for real estate investors. The primary goal is to monitor fluctuations in real estate prices.

For the implementation of this design, I analyzed various CSV files provided by Airbnb, focusing on data regarding property listings, prices, availability, and ratings. The columns to be included in the database tables were selected based on the needs of the two end users.

Chapter 1: Data Warehouse Creation

  1. Database Selection:

    For this project, I chose Microsoft SQL Server Management Studio due to its comprehensive features for data management. After comparing it with alternatives such as PostgreSQL and MongoDB, SSMS was found to be the optimal choice for relational data analysis.

  2. Data Preparation:

    The data was sourced from Airbnb's publicly available CSV files. After downloading, issues with non-displaying Greek characters were resolved by saving the files in UTF-8 format. Unnecessary columns were removed in SSMS using SQL queries.

  3. Data Loading:

    The "Airbnb Listings Greece" database was created in SSMS, and the CSV files were imported, optimizing column data types and handling null values for consistency.

  4. Data Cleaning:

    Non-essential columns (such as reviewer names) were removed. Columns were renamed for clarity (e.g., `id` was renamed to `listing_id`) to ensure consistency across tables.

  5. Table Unification:

    Using the `UNION ALL` operator, regional data was consolidated into three main tables: `calendar`, `listings`, and `reviews`, allowing for a comprehensive analysis of the data.

  6. Fact and Dimension Tables:

    The Data Warehouse was structured with clear separations between fact and dimension tables:

    • Fact Tables: Contain numerical metrics
      • FactReviews
      • FactCalendar
      • FactListings
    • Dimension Tables: Define descriptive data
      • DimHost
      • DimLocation
      • DimPropertyType
      • DimRoomType

    Null values were replaced with default values (e.g., `0`, `1900-01-01`, etc.) and unique IDs were generated for the dimensions using the `IDENTITY` property.

  7. Verification and Optimization:

    Primary keys and relationships were verified to ensure data integrity. Additionally, large files and null replacements were optimized for better performance.

Chapter 2: Tableau Implementation

In my effort to create the dashboards, I used all the tables analyzed in the previous chapter, with the primary focus on the FactListings table. I utilized many different fields from these tables, particularly for the operational dashboard, as it needed to be more detailed. I also made an effort to incorporate various types of charts, each tailored to the specific needs of the analysis. Some of the charts used include pies, bar charts, Gantt charts, treemaps, maps, tables, scatter plots, area charts, or a combination of these. Using the various capabilities offered by Tableau, I was able to effectively visualize and analyze the data. Key features such as clusters, slicers, groups, hierarchies, forecast models, and calculated fields were utilized to enhance the data exploration and insights generation process. These tools allowed for:

  • Clusters: Identifying natural groupings within the dataset to segment listings based on key attributes.
  • Slicers: Enabling interactive filtering to focus on specific subsets of data, facilitating a more detailed and targeted analysis.
  • Groups: Categorizing related data points into meaningful groups to simplify analysis and visualization.
  • Hierarchies: Structuring data in a hierarchical manner to allow for drill-downs, providing deeper insights across different levels of granularity.
  • Forecast Models: Applying advanced forecasting techniques to predict future trends in listing prices, availability, and other relevant metrics.
  • Calculated Fields: Creating custom metrics and aggregations to enhance understanding of the data and support more sophisticated analyses.

This thesis emphasizes the significance of data analysis and visualization in the digital era, focusing on creating a Data Warehouse using Airbnb data to understand market trends. By developing strategic and operational dashboards with Tableau, the project supports informed decision-making for both ministries and investors, enabling them to track market fluctuations and identify investment opportunities. The analysis reveals insights into Airbnb listings across Greece, with the Aegean Islands being the most expensive and popular. It also highlights accommodation quality, pricing patterns, and categorizes listings based on metrics like ratings and revenue, offering valuable information for the growth and regulation of the short-term rental sector in Greece.
